# Next.js & Auth0 Authentication System

This project is a complete authentication and authorization system built with **Next.js (App Router)**, **NextAuth.js**, and **Auth0**. It demonstrates secure user login, JWT-based session management, protected routes using middleware, and role-based access control.

The implementation adheres to modern web development best practices, including **SOLID principles** and the **12-Factor App methodology** for configuration management.

---

## ✨ Features

- **OAuth Authentication**: Secure user sign-in and sign-up via Auth0.
- **Session Management**: JWT-based sessions handled by NextAuth.js.
- **Protected Routes**: Middleware protects specific pages (e.g., `/dashboard`) from unauthenticated access.
- **Role-Based Access Control (RBAC)**: A bonus feature that restricts access to certain pages (e.g., `/admin`) to users with specific roles.
- **12-Factor App Compliant**: All configuration and secrets are managed through environment variables.
- **Dockerized**: Includes a multi-stage Dockerfile for building an optimized production image.

---

## 🛠️ Tech Stack

- **Framework**: Next.js 14+ (App Router)
- **Authentication**: NextAuth.js, Auth0
- **Styling**: Tailwind CSS
- **Language**: TypeScript
- **Deployment**: Docker

## 🚀 Getting Started

Follow these instructions to set up and run the project locally.

### 1. Prerequisites

- Node.js (v18 or later recommended)
- npm, yarn, or pnpm
- An active Auth0 account

### 2. Clone the Repository

```bash
git clone https://github.com/YOUR_USERNAME/next-auth.git
cd next-auth
```

### 3. Install Dependencies

```bash
npm install
```

### 4. Set Up Environment Variables

This project requires credentials from your Auth0 application. Since the .env file is not committed to Git, you need to create it yourself.

- Create a new file named .env.local in the root of the project.

- Copy the contents of the example file below into your .env.local.

- Replace the placeholder values with your actual credentials from the Auth0 dashboard.

`.env.local`

```bash
# Auth0 Provider Credentials
# Get these from your Auth0 Application -> Settings
AUTH0_CLIENT_ID="YOUR_AUTH0_CLIENT_ID"
AUTH0_CLIENT_SECRET="YOUR_AUTH0_CLIENT_SECRET"
AUTH0_ISSUER="https://YOUR_AUTH0_DOMAIN"

# NextAuth.js Configuration
# The NEXTAUTH_SECRET can be generated by running `openssl rand -base64 32` in your terminal
NEXTAUTH_URL="http://localhost:3000"
NEXTAUTH_SECRET="YOUR_GENERATED_SECRET"
```

### 5. Run the Development Server

Once the dependencies are installed and the environment variables are set, you can start the development server.

```bash
npm run dev
```

Open `http://localhost:3000` with your browser to see the result.

## 🐳 Running with Docker

### Build the Docker image:

```bash
docker build -t next-auth-app .
```

### Run the container:

You must pass your environment variables to the container.

```bash
docker run -p 3000:3000 --env-file ./.env.local next-auth-app
```

The application will be available at `http://localhost:3000`
